# Assignment 1 – Design Report

This repository contains the Design Report for Assignment 1 of the *Advanced Data Architectures* course.

## Overview

The objective of this assignment is to design a data-intensive system using Domain-Driven Design (DDD) and modern architectural principles. The system can be implemented as a microservices architecture, a data mesh, a multi-agent system, or a hybrid approach.

The focus is on:

* Identifying business and data domains
* Defining bounded contexts
* Designing services, agents, or data products
* Applying relevant design principles and patterns
* Documenting the architecture in a structured and clear manner

## Report Structure

The Design Report consists of the following sections:

1. **Case Study**
   Description of the system, its context, stakeholders, and high-level functional requirements.

2. **Identification of Domains and Services**
   Application of DDD to define core, supporting, and generic domains. For selected domains, services, agents, or data products are identified and justified using design principles.

3. **Architectural Design**
   Detailed description of the system architecture, including diagrams, components, design decisions, and applied design patterns.

4. **Member Contribution and Reflection**
   Overview of individual contributions and reflection on the design process.

5. **Technology Statement**
   Disclosure of any tools, including AI tools, used during the assignment.

## Requirements

* Apply Domain-Driven Design to structure the system
* Identify and justify services, agents, or data products
* Use at least three design patterns or architectural tactics
* Provide clear diagrams and explanations
* Keep the report within the page limit

## Notes

* The design should be logically consistent and well-justified
* Diagrams can use informal notation (no UML required)
* Generative AI tools may be used, but all outputs must be critically evaluated and documented
